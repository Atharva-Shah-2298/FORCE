# Recovery — parameter recovery against a separated synthetic generator

Forward-model validation of FORCE on N = 5000 voxels with known ground-truth
latents, generated by an **explicitly different generative model** than FORCE's
dictionary. FORCE is run at K = 50 with posterior softmax β ∈ {500, 1000, 2000,
5000, 10000} and compared against AMICO-NODDI, DTI and DKI.

## Design constraints

* **Acquisition match.** Test signals use the same bvals/bvecs as the FORCE
  dictionary (HCP 165840, 288 volumes, shells 0/1000/2000/3000).
* **Compartments.** Stick (intra) + hindered (extra) + GM iso + FW (CSF) iso —
  same compartmental family as the dictionary, no soma and no exchange.
* **Matching.** One K = 50 retrieval per SNR; β reweighting is done over the
  same neighbour set (as in `../scripts/exp5_beta_sweep.py`).

## Generator vs dictionary separation

The test generator deliberately differs from the dictionary in its orientation,
diffusivity and tissue-fraction priors so that recovery is not a self-test:

| Aspect | FORCE dictionary | Test generator |
|---|---|---|
| Orientation dispersion | Bingham, one ODI shared across fibers in a voxel | Watson(μ, κ), κ = 1/tan(π·ODI/2), 64 sticks/fiber, ODI sampled per fiber |
| Diffusivity sampling | Uniform priors over the physical ranges | Truncated Gaussian over the same ranges, independent per compartment |
| Tissue-fraction prior | Dirichlet(2, 1, 1) over (WM, GM, FW) | Logit-normal (softmax of independent Gaussians) |
| Within-WM NDI | Uniform [0.6, 0.9] | Truncated Gaussian μ = 0.55, sd = 0.20 on [0.10, 0.90] |
| Fiber count | K ∈ {1, 2, 3} | K ∈ {0, 1, 2, 3} (10 / 40 / 35 / 15 %) |
| Crossing angle | implicit (min separation) | Truncated Gaussian μ = 55°, sd = 25° on [10°, 90°] |

ND and ODI are compared on FORCE's voxel-wide definitions, applied identically
to the generator latents:

```
ND  = f_WM · NDI_within_WM            (GM, CSF contribute 0)
ODI = 1 − f_WM · (1 − mean_fiber ODI)
```

## Reported metrics

NDI, ODI (voxel), FW, FA, MD, RD — six metrics across 3 SNR × 5 β.

### Headline numbers (FORCE, β = 2000)

```
                       SNR50          SNR20          SNR10
ND  (voxel) bias/MAE   0.020/0.035    0.061/0.068    0.175/0.177
ODI (voxel) bias/MAE   0.101/0.121    0.065/0.120   -0.031/0.132
FW          bias/MAE   0.044/0.063    0.047/0.069    0.038/0.081
FA          bias/MAE  -0.030/0.034   -0.036/0.044   -0.058/0.074
MD ×1e-3    bias/MAE  -0.004/0.014   -0.027/0.035   -0.103/0.105
RD ×1e-3    bias/MAE   0.018/0.026    0.002/0.038   -0.048/0.079
```

FORCE recovers voxel-wide ND tightly at high SNR; a positive Rician-rectification
bias appears at SNR 10 (low-signal voxels look more intra-axonal under magnitude
noise), and AMICO's within-WM v_ic drifts with the same sign. ODI MAE is roughly
flat (~0.12), dominated by the per-fiber Watson vs shared-Bingham difference
rather than noise. β ∈ [1000, 5000] is the stable regime; β = 500 is too soft and
β = 10000 over-sharpens at low SNR.

## Files

```
generate_test_voxels.py   Watson / truncated-Gaussian / logit-normal generator
add_noise_save.py         Rician noise + NIfTI export at three SNRs
run_force.py              FORCE retrieval (1× per SNR) + β reweighting
run_dti_dki.py            DTI + DKI baselines
run_amico.py              AMICO-NODDI baseline
analyze.py                aggregation + figures + text/JSON summary
```

## How to run

```bash
python generate_test_voxels.py --N 5000
python add_noise_save.py
python run_force.py
python run_dti_dki.py
python run_amico.py          # requires the AMICO package
python analyze.py
```

FORCE here is the DIPY implementation (`dipy.reconst.force`); see the
repository root README for installation and the FORCE API.
